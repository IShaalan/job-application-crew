## Role Subtype: Backend Engineer

### What They're Looking For
- Strong fundamentals in data structures, algorithms, and system design
- Experience with distributed systems and high-availability architectures
- Database expertise (SQL, NoSQL, query optimization)
- API design and service-oriented architecture
- Performance optimization and debugging at scale

### Positioning Angles
1. **"Systems thinker"**: Show you design for scale, resilience, and maintainability from day one, not just correctness
2. **"Performance-obsessed"**: Lead with latency reductions, throughput improvements, and cost savings -- not just features shipped
3. **"Reliability champion"**: Demonstrate experience reducing incidents, improving uptime, and building observability into every system

### Keywords
- Distributed systems, microservices, event-driven architecture
- REST, gRPC, GraphQL, API design
- SQL, PostgreSQL, MySQL, Redis, MongoDB, DynamoDB
- Kafka, RabbitMQ, message queues
- Docker, Kubernetes, AWS/GCP/Azure
- CI/CD, infrastructure as code, Terraform
- Performance profiling, load testing, caching
- Data modeling, query optimization, indexing

## Role Subtype: Frontend Engineer

### What They're Looking For
- Mastery of modern JavaScript/TypeScript frameworks (React, Vue, Angular, Svelte)
- Deep understanding of browser performance and web vitals
- Accessibility knowledge (WCAG compliance)
- Component architecture and design system experience
- Strong collaboration with design teams

### Positioning Angles
1. **"User experience engineer"**: Bridge the gap between design intent and technical implementation -- show you care about the end-user experience, not just the code
2. **"Performance-first builder"**: Lead with Core Web Vitals improvements, bundle size reductions, and perceived performance gains
3. **"Design system architect"**: Show you've built reusable component libraries that scale across teams and reduce development time

### Keywords
- React, Vue, Angular, Svelte, Next.js, Remix
- TypeScript, JavaScript, HTML, CSS
- Web performance, Core Web Vitals, Lighthouse
- Accessibility, WCAG 2.1, ARIA
- Design systems, component libraries, Storybook
- State management, Redux, Zustand, MobX
- Testing (Jest, Cypress, Playwright)
- Webpack, Vite, build optimization
- Responsive design, mobile-first, PWA

## Role Subtype: Full-Stack Engineer

### What They're Looking For
- Ability to own features end-to-end (database to UI)
- Pragmatic technology choices (right tool for the job, not resume-driven)
- Strong debugging skills across the entire stack
- Understanding of both frontend and backend performance
- Comfort with rapid iteration and shipping

### Positioning Angles
1. **"End-to-end owner"**: Show you can take a feature from database schema to polished UI without handoffs -- emphasize autonomy and shipping speed
2. **"T-shaped engineer"**: Demonstrate depth in one area (backend or frontend) while being dangerous in the other -- avoid looking like a generalist who is mediocre at both
3. **"Startup-speed builder"**: For startup roles, lead with shipping velocity and pragmatic trade-offs. For growth-stage, lead with scalable architecture decisions

### Keywords
- Full-stack, end-to-end
- React/Vue + Node.js/Python/Ruby/Go
- REST API, GraphQL
- PostgreSQL, MongoDB, Redis
- Docker, CI/CD, deployment
- Testing (unit, integration, E2E)
- Agile, sprint planning, code review
- Cloud services (AWS, GCP, Azure)

## Role Subtype: Platform / Infrastructure Engineer

### What They're Looking For
- Experience building internal tools and developer platforms
- Deep understanding of CI/CD, deployment pipelines, and developer workflows
- Cloud infrastructure expertise (IaC, multi-region, cost optimization)
- Observability and monitoring at scale
- Ability to measure and improve developer productivity

### Positioning Angles
1. **"Developer productivity multiplier"**: Frame everything in terms of how many engineers you unblocked and how much faster the org ships as a result
2. **"Reliability at scale"**: Lead with uptime numbers, incident reduction, and MTTR improvements -- these are the metrics platform teams live by
3. **"Migration expert"**: Show you've successfully moved systems (cloud providers, CI tools, container orchestration) with measurable adoption and zero downtime

### Keywords
- Platform engineering, developer experience (DX)
- Kubernetes, Docker, container orchestration
- Terraform, Pulumi, CloudFormation, IaC
- CI/CD, GitHub Actions, Jenkins, ArgoCD
- Observability, Datadog, Grafana, Prometheus, OpenTelemetry
- AWS, GCP, Azure, multi-cloud
- Service mesh, Istio, Envoy
- MTTR, SLOs, SLIs, error budgets
- Developer velocity, DORA metrics
- Cost optimization, FinOps

## Role Subtype: ML Engineer

### What They're Looking For
- Strong software engineering fundamentals applied to ML systems
- Experience building and maintaining ML pipelines (training, serving, monitoring)
- Model deployment and serving at scale
- Feature engineering and data pipeline expertise
- Collaboration with data scientists and product teams

### Positioning Angles
1. **"ML systems builder, not just model trainer"**: Emphasize production ML infrastructure (serving, monitoring, retraining) over notebook experiments -- show you build systems that run reliably, not just prototypes
2. **"Bridge between research and production"**: Show you can take a data scientist's prototype and turn it into a production system with proper testing, monitoring, and rollback capabilities
3. **"Data pipeline architect"**: Lead with data quality, feature stores, and training pipeline reliability -- models are only as good as the data and infrastructure behind them

### Keywords
- ML pipelines, MLOps, model serving
- TensorFlow, PyTorch, JAX, scikit-learn
- Feature engineering, feature stores
- Model monitoring, drift detection, A/B testing
- Kubeflow, MLflow, SageMaker, Vertex AI
- Data pipelines, Apache Spark, Airflow, dbt
- Vector databases, embeddings, RAG
- GPU optimization, distributed training
- Python, SQL, Scala
- Docker, Kubernetes, cloud ML services

## Cross-Cutting Guidance

### Resume Structure for Engineers
- **Summary**: 2-3 sentences. Years of experience + primary tech stack + scale indicators (users, requests/sec, team size). No adjectives.
- **Key skills/technologies**: Ordered by JD relevance, not alphabetical. Group logically (Languages | Frameworks | Infrastructure | Data).
- **Experience bullets**: Lead with the impact, not the technology. "Reduced API latency by 60%" beats "Used Redis for caching."
- **Side projects / OSS**: Include if relevant to the role. Shows passion and initiative, especially for startups.

### Bullet Writing Rules
- Start with a strong action verb (Architected, Built, Designed, Led, Migrated, Optimized, Reduced, Shipped)
- Include a metric in every bullet (latency, throughput, cost, adoption rate, incident reduction)
- Keep bullets under 25 words
- End every bullet with a period
- Avoid: "Responsible for", "Helped with", "Worked on"

### Common Mistakes
- **Listing technologies without impact**: "Used Kubernetes and Terraform" tells the HM nothing. Say what you did with them and what changed.
- **Missing scale indicators**: "Built a service" vs "Built a service handling 50K req/s across 3 regions" -- scale is the differentiator.
- **No mention of collaboration**: Engineering is a team sport. Show you work with product, design, and other engineers.
- **Resume reads like a job description**: Bullets should be achievements, not responsibilities.

### Cover Letter Tips for Engineers
- Open with a specific technical challenge you solved, not "I am writing to apply"
- Connect your technical work to business impact
- Show you understand the company's technical challenges (reference their tech blog, open source, or product)
- 200-300 words, 3 paragraphs max
- Engineers are often forgiven for shorter, more direct cover letters
